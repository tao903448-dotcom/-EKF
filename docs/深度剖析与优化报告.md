# 自适应 EKF 系统 —— 深度剖析与优化报告

> 第十五届中国软件杯 · A8 四旋翼无人机位姿控制系统
> 版本：重构版 / 2026-06-20
> 范围：仓库全量源码（矩阵库 + EKF 框架 + 测试 + 演示）

---

## 0. 一句话结论

**原项目的核心算法库存在一个会让"标准 EKF"协方差坍缩为零的致命 bug，加上演示程序里算法标签错位，导致文档中"自适应 EKF 使 RMSE 降低 65%"这一卖点，主要是 bug 制造的假象，而非算法本身的功劳。** 本次重构修复了全部已定位缺陷，并用一个**诚实的双场景基准**重新证明：鲁棒/自适应方法确实有价值——但只在它们对症的场景（测量野值）下，且这一点必须如实呈现。

## 2. 致命与重要缺陷（按严重度排序，均有实证）

### 🔴 B1：矩阵乘法别名不安全 → 标准 EKF 协方差坍缩为 0
`matrix_mul` 先 `memset(result)` 再读取输入。当 `result` 与某个输入同址时（EKF 标准更新里 `matrix_mul(&P, &temp2, &P)` 计算 `P=(I-KH)P`），它**先把 P 清零再去乘**，于是 P 恒等于 0。

后果：标准 EKF 一步更新后协方差归零 → 卡尔曼增益 K→0 → 滤波器从此**无视所有观测**，状态被冻结在初值附近。

实证（一步 predict+update，2 维匀速模型）：
```
P after predict : [[2.01, 1.00],[1.00, 1.01]]      ← 正确
P after update  : [[0, 0],[0, 0]]   trace=0         ← 原始：坍缩
P after update  : [[0.222,0.111],[0.111,0.567]]     ← 修复后：与手算一致
```

同类别名缺陷还潜伏在 Joseph / Student-t / 自适应里的 `K R K'` 项（`matrix_mul(&temp1,&temp1,&K_T)`），同样会被清零。

**修复**：在 `matrix_mul` / `matrix_transpose` / `matrix_inverse` 内部检测 `result` 与输入别名，自动走临时缓冲——一处根因修复，覆盖所有调用点。

### 🔴 B2：矩阵求逆缓冲区越界（栈溢出）
`matrix_inverse` 把 `n×2n` 的增广矩阵塞进 `Matrix.data[256]`。当 `n ≥ 12` 时 `n·2n > 256`，**越界写栈**。EKF 观测维度上限为 16，可触发。

实证：对原始 `matrix.c` 跑 14×14 求逆，程序 `*** stack smashing detected ***` 直接崩溃（栈金丝雀被覆盖）。

**修复**：增广矩阵改用独立的 `float aug[16*32]` 缓冲，并让结果最后写出（兼容 `result==mat` 别名）。

### 🟠 B3：预测步状态函数输入/输出混叠
`ekf_predict` 调用 `state_func(&x, u, &x, dt)`，把同一个 `x` 既当输入又当输出，违反头文件"输入/输出分离"的约定。对一般非线性 `f`（先写 `x_new[0]` 后读 `x[1]`）会读到已被覆盖的旧状态。

实证：用"交换"状态函数 `x_new=[x1,x0]`，原实现得 `[7,7]`（错），修复后得 `[7,3]`（对）。

**修复**：`EKF_State` 增加独立的 `x_pred` 缓冲。

### 🟠 B4：Student-t 鲁棒权重方向写反
原式 `R_scaled = R * (ν+n)/(ν+d²/δ²)`：野值（马氏距离 d 大）→ 权重小 → R **缩小** → 增益**变大** → 对野值更信任，与"鲁棒"背道而驰。

**修复**：改为 `R_eff = R * (ν + d²/δ²)/(ν + n)`，野值 → R 膨胀 → 增益收缩 → 正确降权。

### 🟠 B5：NEON 版本构建必然失败
`matrix_neon.c` 重定义了 `matrix.c` 的 5 个核心函数（加/减/乘/转置/数乘），却**缺失** `matrix_init/eye/inverse/copy/cholesky/...` 等约 18 个函数。Makefile 的 ARM 目标链接 `matrix_neon.o + ekf.o`，会因大量未定义符号而失败；若改为同时编两份又会重复符号。即"ARM NEON 优化"实际从未能构建。

**修复**：删除 `matrix_neon.c`，把 NEON 内核以 `#if __ARM_NEON` **内联进唯一的 `matrix.c`**。x86 自动走标量、ARM 自动走 SIMD，无重复符号、无缺函数，ARM 构建真正可用。

### 🟠 B6：演示程序算法标签错位
GUI 里两处映射不一致：
- `InitEKF` 用 `0→标准,1→Student-t,2→自适应,3→Joseph`；
- `run_ekf_algo` 用 `(EKF_UpdateMethod)method` 直接强转，得到 `0→标准,1→Joseph,2→Student-t,3→自适应`。

二者都与库枚举顺序 `STANDARD=0,JOSEPH=1,STUDENT_T=2,ADAPTIVE=3` 不一致。结果："性能对比"四根柱子的标签与其真正运行的算法对不上，显示轨迹用的算法也和同名柱子不是一个。

**修复**：抽出唯一映射函数 `UiToMethod()`，`InitEKF` 与 `run_ekf_algo` 统一调用；并删除 `RunAllAlgorithms` 里重复的一段仿真循环。新增 `test_ekf_enum_order` 守卫枚举顺序。

### 🟡 其他
- 自适应更新只取 `y.data[0]/S.data[0]`（仅 1 维有效）→ 改为多维归一化新息平方 NIS。
- 每步更新后未对协方差对称化 → 新增 `(P+P')/2`，抑制浮点不对称漂移。
- `matrix_determinant` 用 O(n²) 遍历置换矩阵定符号 → 改为 LU 中直接累计行交换符号。
- 四个更新函数近乎复制粘贴（各 ~60 行）→ 抽取 `ekf_innovation_cov / ekf_gain / ekf_apply_state / ekf_cov_joseph` 辅助函数。
- 文档/README 引用了**根本不存在**的文件（`build.bat`、`examples/ekf_demo.c`、`docs/DEVELOPMENT_STATUS.md`、`自适应EKF系统.exe`）→ 已补齐 `ekf_demo.c` 并修正文档。

## 3. 关键洞察：被 bug 制造的"算法优越性"

文档宣称"自适应 EKF RMSE 降低 65%、Student-t 降低 43%"。复现演示的场景（1 维常值模型追踪快速正弦，Q=1,R=30）：

| 阶段 | 标准 | Student-t | 自适应 | Joseph |
|---|---|---|---|---|
| 原始（含 bug，对齐文档≈42.7/24/16.7/46.6） | 43.2 | 25.0 | 16.0 | 47.0 |
| **仅修复 B1（标准更新别名）** | **21.1** | 25.0 | 16.0 | 47.0 |
| 全部修复 + 正确枚举 | 21.1 | 46.1 | 45.6 | 21.1 |

要点：
1. **仅改一行**（修标准更新的协方差别名），标准 EKF 就从 43 降到 21——所谓"标准 EKF 很差"是协方差坍缩 bug 把它冻结在初值造成的假象。
2. 用**正确的枚举**后，`Standard == Joseph = 21.1`（两者对最优增益数学等价，这条恒等式正好反过来证明实现已正确）。
3. 在这个场景里，Student-t / 自适应反而更差——因为问题是**过程模型失配**（常值模型追不上正弦），应增大 Q；而这两种方法靠"膨胀 R、压低增益"工作，方向不对。

**所以原"降低 65%"既源于标准路径的 bug，也源于用错了基准场景。**

## 4. 诚实的双场景评测（重构后）

新命令行 demo (`examples/ekf_demo.c`) 同时跑两类场景：

**场景 A —— 过程失配（常值模型追正弦，无野值）**
```
标准EKF    RMSE = 15.7
Joseph     RMSE = 15.7   (≡标准)
Student-t  RMSE = 45.0
自适应EKF  RMSE = 47.6
```
→ 标准最优；鲁棒/自适应对过程失配无能为力。

**场景 B —— 测量野值（匀速模型 + 15% 脉冲野值，模型正确）**
```
标准EKF    RMSE = 5.98
Joseph     RMSE = 5.98   (≡标准)
Student-t  RMSE = 0.64   ↓ 89%
自适应EKF  RMSE = 0.76   ↓ 87%
```
→ 这才是鲁棒滤波真正的主场，降幅与原文档量级相当，但**这次是真的**。

**结论：不存在"永远最优"的更新方法，必须按噪声特性选型。** 这是比"自适应永远最好"更有价值、也更经得起评委追问的表述。

## 6. 仍存在的局限与后续建议

1. **GUI demo 的装饰性假数据**：遥测页的 `COV: 0.124`、`50.2 Hz`、`GNSS Precision Fix`、经纬度等为硬编码，应接真实滤波器内部量或明确标注为占位。
2. **观测噪声生成**：GUI 用查表均匀分布却标称 "Gaussian(0,0.5)"；CLI demo 已改用 Box-Muller 真高斯。
3. **模型与课题的匹配**：课题是四旋翼位姿，但 demo 用的是 1 维常值/匀速模型。建议升级为含姿态角的多维模型，让"位姿控制"名副其实。
4. **自适应方向**：当前自适应只调 R；针对"模型失配"主题，调 Q（过程噪声）往往更对症，可作为增强项。
5. **定点化 / MISRA**：嵌入式落地可进一步评估定点实现与 MISRA-C 合规。

# 四旋翼姿态估计 EKF —— 实验报告

> 第十五届中国软件杯 · A8 四旋翼无人机位姿控制系统设计优化
> 版本：2026-06-21 ｜ 复现：`make run-attitude`

---

## 1. 问题与定位

A8 题目的核心是**四旋翼无人机位姿估计**。本项目据此实现一套基于 IMU
（陀螺 + 加速度计 + 磁力计）的姿态估计 EKF，并把项目的算法创新——
**面向模型失配的自适应 / 鲁棒滤波**——直接作用于该真实问题，定量回答：
*在振动野值、机动失配等真实干扰下，自适应 / 鲁棒更新能带来多少收益？*

## 2. 滤波器建模

| 项 | 设计 |
|---|---|
| 状态 x (7) | 姿态四元数 q（body→world, Hamilton）+ 陀螺零偏 b (rad/s) |
| 控制 u (3) | 陀螺测量 ω_m (rad/s) |
| 预测 | q ← q ⊗ Δq(​(ω_m−b)·dt)，b 随机游走；每步重归一化(框架 state_normalize 钩子) |
| 观测 z (6) | 加速度计"重力方向"单位向量 + 磁力计"参考方向"单位向量（均 body 系） |
| 观测模型 | h(x) = [R(q)ᵀ·ĝ_w ; R(q)ᵀ·m̂_w] |
| 雅可比 | 中心差分**数值雅可比**（correct-by-construction，避免手推易错） |

设计取舍：
- **四元数**避免欧拉角万向锁，适合大机动；采用"加性 EKF + 重归一化"以
  复用通用框架的全部更新方法（严格乘性 MEKF 误差状态形式列为后续增强）。
- **方向型观测**：加速度计在机动（比力≠重力）时，其"重力方向"天然偏离真值，
  恰好构成**模型失配**——这正是自适应 / 鲁棒更新的考点。

## 3. 实验设置

- 仿真：可复现 IMU 仿真器（`include/imu_sim.h`，确定性伪随机），200 Hz，15 s。
- 蒙特卡洛：每格 20 个随机种子取平均。
- 噪声：陀螺白噪声 ~0.57°/s + 零偏随机游走；加速度/磁方向噪声 σ≈0.02。
- 标定：观测噪声 R = 4×10⁻⁴（与真实方向噪声方差一致）。
- 三场景：
  1. **CLEAN**：温和机动、无野值；
  2. **OUTLIER**：注入 12% 加速度脉冲野值（电机振动）；
  3. **MANEUVER**：大机动 + 6 m/s² 线加速度（比力严重偏离重力）。
- 指标：稳态**姿态 RMSE（度）**、陀螺**零偏 RMSE**、平均 **NIS**（新息一致性）、
  **NIS 越界比例**（超 95% χ² 上界）。
  *注：单位方向观测受单位范数约束，每个 3 维观测有效自由度≈2，两个观测合计≈4，
  故一致滤波的平均 NIS 应≈4 而非 6。*

## 5. 分析

1. **CLEAN**：四法接近，姿态 RMSE≈0.42°，平均 NIS≈3.8≈有效自由度，**滤波器一致**；
   `标准 ≡ Joseph`（对最优增益数学等价）佐证实现正确。
2. **OUTLIER**：标准 EKF 被振动野值污染，RMSE 升至 1.75°、NIS 飙到 73（严重过自信、
   不一致）；Student-t / 自适应通过对大马氏距离观测降权，**RMSE 降低 76–79%**，
   并把 NIS 拉回≈2–4（**恢复一致性**）。这说明收益不仅在精度，更在估计的可信度。
3. **MANEUVER**：线加速度使加速度计"重力方向"持续失配，标准 EKF 误差达 25.7°；
   Student-t **↓54%**、自适应 **↓46%**；叠加加速度门控(§5.1)后 **7.67°（↓70%）**。
   加速度计在持续线加速度下本质上失去重力信息，仅靠陀螺(漂移)+磁力计(偏航)
   无法完全补偿，故仍有残差——这是**诚实的物理上限**，进一步改善需引入
   运动模型 / GPS 等外部信息（后续工作）。

**结论：不存在"永远最优"的更新方法；自适应 / 鲁棒更新在其对症场景
（测量野值、观测失配）下能显著且可验证地提升四旋翼姿态估计的精度与一致性。**

### 5.1 加速度自适应门控（acceleration rejection）

针对 MANEUVER 这一最弱项，进一步引入物理先验：当加速度计比力幅值 |f| 偏离
重力 g 时，说明其受线加速度/野值污染、所给"重力方向"不可信，于是按偏差
`gate = 1 + k·(|f|/g − 1)²` 放大加速度观测噪声块（磁块不变）。20 种子蒙特卡洛：

| 场景 | 方法 | 无门控 RMSE(°) | +门控 RMSE(°) | 增益 |
|---|---|--:|--:|--:|
| CLEAN | 标准/自适应 | 0.42 / 0.41 | 0.42 / 0.41 | ≈0（不劣化） |
| OUTLIER | 标准 | 1.75 | 0.68 | ↓61% |
| MANEUVER | 标准 | 25.7 | 22.5 | ↓12% |
| MANEUVER | 自适应 | 13.8 | **7.67** | **↓44%** |

门控按比力物理量"对症"加权：CLEAN 下 |f|≈g、门控≈1 几乎不动；机动/野值下
自动降权加速度，转而依赖陀螺+磁力计。**自适应 + 门控**叠加给出 MANEUVER 最优
**7.67°（相对标准 EKF 25.7° 累计 ↓70%）**。仍存在残差是机动期重力信息缺失的物理
上限，彻底解决需运动模型/外部定位（路线图）。

### 5.2 参数寻优（算力服务器 64 核蒙特卡洛网格搜索）

`examples/param_sweep.c`（OpenMP）在 64 核算力服务器上对关键参数做大规模
多种子（80 seeds）网格寻优，2.2 s 完成 ~5000 次滤波运行：

- **OUTLIER / Student-t (ν, δ)**：δ=0.5 在各 ν 下均最优；选 ν=4, δ=0.5
  （OUTLIER 0.44→0.37，且 CLEAN 同时改善到 0.36）。
- **MANEUVER / 自适应+门控 (α, K)**：误差随 α、K 单调下降；为避免过拟合并兼顾
  泛化，做**跨场景权衡表**：

  | 配置 | CLEAN | OUTLIER | MANEUVER |
  |---|--:|--:|--:|
  | 原默认 α=1,K=200 | 0.408 | 0.416 | 12.49 |
  | **采用 α=8,K=800** | 0.408 | 0.416 | **7.67** |
  | 极端 α=16,K=1600 | 0.411 | 0.418 | 6.74 |

  关键发现：**加大降权对 CLEAN/OUTLIER 几乎无影响**——因门控/自适应"按需触发"
  （CLEAN 下 NIS≈4<6 故因子≈1、|f|≈g 故门控≈1），仅在真失配时起作用。故可
  放心采用 α=8, K=800：MANEUVER 大降而良性场景不退化，无实质权衡。
  （这正是自适应/鲁棒方法的设计要义：不需要时休眠。）

## 7. 工程要点

- 姿态模型作为通用 EKF 框架的一个配置（`src/attitude.c`），四种更新方法零成本复用；
- 为带约束状态新增 `state_normalize` 钩子（四元数重归一化），框架更通用；
- 数值雅可比使复杂非线性模型免于手推，降低出错面；
- GitHub Actions CI 六作业（gcc / clang / -Werror 严格 / ASan-UBSan / ARM(NEON)
  交叉编译 / cppcheck 静态分析），全绿；服务器侧另跑 gcov 覆盖率(总 78.8%)。
- **实时性**：自适应 EKF 单步(predict+update) 约 6 µs（x86-64 -O2），可支撑约
  168 kHz，远高于 200 Hz 飞控更新率（`make run-attitude` 末尾实测）。
- **核心新息卡方门控**（`ekf_set_nis_gate`）：按 NIS 超阈值整步拒绝野值，对所有
  更新方法通用；OUTLIER 下仅此一项即让标准 EKF 姿态 RMSE 1.94°→0.40°。
